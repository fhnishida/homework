Exercise 3.3.¶

Using the data on consumption and personal disposable income in Canada for the period 1947 to 1996 in the file consumption.data, estimate the model¶

$$c_t = \beta_1 + \beta_2y_t+u_t,\ \ \ \ \ u_t\ \sim \ iid\ N(0,\sigma^2) $$

where $c_t=\log{C_t}$ is the log of consumption and $y_t=\log{Y_t}$ is the log of disposable income, for the entire sample period. Then use the estimates of $\beta_1,\beta_2$, and $\sigma$ to obtain 200 simulated observations on $c_t$. Begin by regressing your simulated log consumption variable on the log of income and a constant using just the first 3 observations. Save the estimates of $\beta_1,\beta_2$, and $\sigma$. Repeat this exercise for sample sizes of $4, 5, ..., 200$. Plot your estimates of $\beta_2$ and $\sigma$ as a function of the sample size. What happens to these estimates as the sample size grows? Repeat the complete exercise with a different set of simulated consumption data. Which features of the paths of the parameter estimates are common to the two experiments, and which are different?¶

We'll load "consumption.data.txt" provided by the authors in http://qed.econ.queensu.ca/ETM/data/, define the number of simulations $B=200$, and create a vector to fill it when performing the simulations.

Our first simulation is using the exact order of the consumption dataset's observations and performing regression with selected observations to save $\beta_1, \beta_2 \text{ and } \sigma$ estimates in $comparison$ vector, that is empty yet. We begin by selecting the first three observations, for each time we run a simulation, we increase one more observation (always follow the defined order) until we reach a regression with 200 observations, that is, reaching 198 regressions.

#First Simulation - Simulation by original observations order
for (i in 3:B){
  reg = lm(log(consumption[1:i]) ~ log(income[1:i]), data) #; summary(reg)
  coeff = summary(reg)$coefficients
  
  #Saving the regression estimates and standard deviation
  comparison[i-2,"sample_size"] = i
  comparison[i-2,"beta_1_sim1"] = coeff["(Intercept)", "Estimate"]
  comparison[i-2,"beta_2_sim1"] = coeff["log(income[1:i])", "Estimate"]
  comparison[i-2,"sigma_sim1"] = summary(reg)$sigma 
}

The second simulation is almost the same as the first. We'll begin by sorting arbitrarily the dataset exact order, resulting in the same dataset, but the observations will almost surely be in different rows, that is, having a new order of the observations. $$$$ Then, we'll also perform the regression and saving the estimates in $comparison$ vector. We also begin the three observations and we'll continue to add one variable, following the new arbitrarily order, in each simulation. For example: $$$$ $\underline{\text{In Simulation 1}}$: we'll begin by running the regression with thre first observations numbers 1, 2 and 3; the following regression will have 4 observations: numbers 1, 2, 3 and 4; until reaching the regression with all 200 observations. $$$$ $\underline{\text{In Simulation 2}}$: we'll begin by running the regression with three arbitrarily observations, say, numbers 20, 175 and 42; the following regression will also have the past 3 observations + 1 arbitrarily one, say, numbers 20, 175, 42 and 66; until reaching the regression with all 200 observations.

Note that, the estimates vary wildly at the beginning and then settle down, although $\sigma$ estimates settle down less quickly than did the estimates of $\beta_2$. The reason for this is that $y_t$ is a trending variable. This contradicts assumption (3.17), used in Section 3.4 when discussed the rate of convergence of $\hat\beta$. $$$$ As a consequence, $\sum^n_{t=1} y^2_t$ grows at a rate much faster than $n-1$, and $\hat\beta_2$ therefore converges at a rate considerably faster than $n^{−1/2}$.
